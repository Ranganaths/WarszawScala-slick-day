Let's talk about what's wrong with ORM?

Object-relational impedance mismatch
(The Vietnam of Computer Science)

  • Leaky abstraction. You can feel those are not simple objects
  • Very easy to start, very hard to bend it to all needs
  • Complex, complicated, sometimes performance killer
  • Often used incorrectly:
    • Getting the whole object-graph from database
    • Referring to objects not fully fetched, ouside of the persistence context

How Slick differs?

  • A Functional Relational Mapper for Scala - it's all about composing stuff
  • Does not try to mimic DB relations as objects
  • Uses tuples
  • You already know how to write code in it (collections-like API, for-comprehension FTW!)
  • Flexible query composition
  • It's very specific when it hits the database so it's easier to track side-effects
  • It's type safe
  • Embraces SQL and maps it to Scala features

Working with Slick

Schema definition

import slick.driver.H2Driver.api._ //or any other driver

class Artist(tag:Tag) extends
Table[(Long,String)](tag,"ARTISTS")  {
  def id : Rep[Long] =
  	column[Long]("ARTIST_ID",O.PrimaryKey,O.AutoInc)
  def artistName : Rep[String] =
  	column[String]("ARTIST_NAME")
  def * : ProvenShape[(Long,String)] = (id,artistName)
}

class Place(tag:Tag) extends
Table[(Long,String)](tag,"PLACES") {
  def id : Rep[Long] =
  	column[Long]("PLACE_ID",O.PrimaryKey,O.AutoInc)
  def placeName : Rep[String] =
  	column[String]("PLACE_NAME")
  def * : ProvenShape[(Long,String)] = (id,placeName)
}

class Show(tag:Tag) extends

  def artistId : Rep[Long] = column[Long]("ARTIST_FID")
  def placeId : Rep[Long] = column[Long]("PLACE_FID")
  def * : ProvenShape[(Long,Long)] = (artistId,placeId)

  def author = foreignKey("ARTIST_FK",artistId,TableQuery[Artist])(_.id)

How to combine all of what was said?

  • Slick uses a DBIO[T] abstraction for actions
  • This is done for seperation of WHAT is to be done from HOW it is done and WHERE it is to be done
    • Driver abstraction
    • Database abstraction
    • Query composition abstraction

Generating database scheme with initial data


val setupAction: DBIO[Unit] = DBIO.seq(
      (artists.schema ++ places.schema ++ shows.schema).create,
      artists += ( -1, "Captain Jack"),
      artists += ( -1, "Bob Marley"),
      artists += ( -1, "Celine Dion")
    )
var db: Database = Database.forConfig("mydb")
val r : Future[Unit] = db.run(setupAction)
